نموذج OSI المرجعي

"رحلة البيانات من التطبيق إلى السلك"

7. التطبيقات (Application)

الوظيفة: واجهة المستخدم المباشرة للشبكة.

HTTP, HTTPS, FTP, DNS

6. التقديم (Presentation)

الوظيفة: التشفير، الضغط، وصياغة البيانات.

SSL, JPEG, ASCII, MPEG

5. الجلسة (Session)

الوظيفة: فتح وإدارة وإنهاء الاتصال بين الأجهزة.

RPC, NetBIOS, PPTP

4. النقل (Transport)

الوظيفة: تقطيع البيانات وضمان وصولها (Reliability).

TCP (موثوق), UDP (سريع)

3. الشبكة (Network)

الوظيفة: العنونة المنطقية IP واختيار المسار (Routing).

IPv4, IPv6, ICMP | Device: Router

2. ربط البيانات (Data Link)

الوظيفة: العنونة الفيزيائية MAC واكتشاف الأخطاء.

Ethernet, ARP | Device: Switch

1. الفيزيائية (Physical)

الوظيفة: نقل البتات (Bits) كإشارات عبر الوسائط.

Cables, Hub, Wireless Signals

بروتوكول الإنترنت IP Address

تمارين تطبيقية وفهم قناع الشبكة

القاعدة الذهبية: 255 = جزء الشبكة ثابت، 0 = جزء الجهاز متغير.

A

مثال الفئة A

IP: 10.50.20.1
Mask: 255.0.0.0

عنوان الشبكة: 10

عنوان الجهاز: 50.20.1

ملاحظة: أول رقم (10) محجوز للشبكة والباقي متاح لتوزيع ملايين العناوين.

B

مثال الفئة B

IP: 172.16.5.100
Mask: 255.255.0.0

عنوان الشبكة: 172.16

عنوان الجهاز: 5.100

ملاحظة: أول رقمين محجوزان للشبكة والباقي (65,534 عنوان) للأجهزة.

C

مثال الفئة C

IP: 192.168.1.15
Mask: 255.255.255.0

عنوان الشبكة: 192.168.1

عنوان الجهاز: 15

ملاحظة: أكثر الفئات شيوعاً في المنازل، توفر 254 عنواناً فقط للأجهزة.

مستقبل العنونة: IPv6
